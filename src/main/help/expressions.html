Literals

A literal is an entity whose value is expressed in the characters that represent it. For example, 24 is the literal representation of the value 24, "abc" is the literal representation of the character string 123, and [observer velocity 0.5] is the literal representation of an observer with a specific worldline.

Literal values are covered in the section on values.

Variables

Variables are named entities that can be thought of as an address to a value. A value can be stored or fetched using the variable's name. Variables receive values through assignment statements and they can be used in expressions anywhere that a literal is allowed.

A variable starts with an alphabetic character. This character can be followed by any number of letters, digits, or underscores.

Variables are created when they are first assigned a value. The type of variable is the same as the type of the value assigned to it. This type could change in a future assignment.

Static Variables

When a script is first loaded, there are no variables defined; variables are created as they are assigned their first value. Under some circumstances (for example, animation) a script may be re-executed without being re-loaded. For most variables, there is no difference between the first execution (when the script is loaded) and any subsequent execution—the script begins with a clean slate.

Static variables are one exception; once created, they retain their values across script executions (but not across script loads). Once a variable is marked as static, it remains static and can be used like any other variable.

To initialize a static variable only once requires knowing if it exists. The defined() function allows you to find out without generating an error if the variable doesn't yet exist.

Dynamic Variables

Dynamic variables are another type of variable that persists across multiple executions. You initialize these just once using a special assignment statement. This assignment statement is ignored when a script is re-executed, so the defined() function is not needed. Any further attempt to assign a dynamic variable a value within the same execution will produce an error.

Each time a script is re-executed, Gamma may change the value of a dynamic variable based on the variable's type and the way you defined it.

Animation Variables

Animation variables' values are controlled by the animation engine. Their values are incremented or decremented based on the animation frame.

Display Variables

Display variables' values are set by user interface controls manipulated by the end user. Any change to a control causes the script to re-execute. The display variable will receive the control's new value.

Arrays

Array variables are variables immediately followed by an expression enclosed in brackets.

If the expression evaluates to a float, it is rounded towards 0 to an integer. The integer is converted to a string, prefixed with a dollar sign, and appended to the variable's name.

If the expression evaluates to a string, the string is prefixed with a dollar sign and appended to the variable's name.

This creates a new name. Other than having the name created dynamically, the variable operates like any other variable. Note that it is impossible to create these variable names directly since the dollar sign is not valid in variable names. Note also that, following the rules above, a[1] would be the same as a["1"].

Constants

Some variable names are reserved to hold specific, constant values. In general, constants are written in upper-case, with words separated by underscores, but the most common constants exist in both upper- and lower-case forms. The constants are:

  • NULL or null — A special value that means the absence of a value.
  • TRUE or true — 1.0.
  • FALSE or false — 0.0
  • INF or inf — positive infinity (for negative infinity, use -INF or -inf).
  • PI — The value of pi.
  • E —The base of the natural logarithms.

Operators

Operators operate on one or two values to create a new value. The types of values supported depends on the operators.

Precedence

When we have a mix of operators, as in 5 + 4 * 4, we need to know if this will be processed as (5 + 4) * 4 or 5 + (4 * 4). Each operator has a precedence value; the smaller the value, the earlier the operator is processed. In the case above, the multiplication operator ("*") has a precedence of 4, while the addition operator has a precedence of 5, so 5 + 4 * 4 becomes 5 + (4 * 4).

You can control the precedence by using parentheses, as in the example above.

Associativity

When operators have equal precedence, what order are they processed in? Does 10 / 2 / 5 become (10 / 2 ) / 5 or 10 / (2 / 5)? If the operators have left-associativity, the expression becomes (10 / 2) / 5. Only a few operators have right associativity. For example, 2^3^4 becomes 2^(3^4).

Data Types

Some operators only work on certain data types. We'll highlight a few here:

  • For the + operator, if both values are floats, the result is a float. If one value is a string, the other value can be anything. The value will be converted to its string representation and concatenated on to the string. For example, (2,5) + " event 1" becomes the string "(2, 5) event 1".
  • The "<-" and "->" perform Lorentz transformations. (2, 5) <- frame1 transforms the coordinate (2, 5) relative to inertial frame "frame1" to the rest frame. The "->" goes the other way: (2, 5) -> frame1 transforms rest coordinate (2, 5) to the "frame1" inertial frame.

Functions

In Gamma, functions return single values, although that value can be a multivalued object. The following functions are available:

Lorentz Transformation Functions

Note that the Lorentz transformation are performed using the -> and <- operators.

Function Returns
gamma( expr ) The gamma value for the given value. The value can be a float (velocity) and frame or an observer promoted to a frame (at tau = 0).
gammaToV( float ) The velocity matching the given gamma. The sign of the velocity matches the sign of the gamma value.

Angle Functions

Function Returns
toXAngle( float ) The angle of the x-axis of an inertial frame with the given velocity. The angle is in degrees from -45° (inclusive) to +45° (inclusive).
toVAngle( float ) The angle of the x-axis of an inertial frame with the given velocity. The angle is in degrees from -90° (exclusive) to -45° (exclusive) or from +45 (exclusive) to +90° (inclusive).
toRelativeAngle ( float , expr ) The first value is an angle in degrees relative to the rest frame. The second value can be a float (velocity) and frame or an observer promoted to a frame (at tau = 0). The function returns the equivalent angle in the frame specified by the second argument. The angle is in the range -180, exclusive, to 180, inclusive. Note that if toRelativeAngle(angle1, v) = angle2, then toRelativeAngle(angle2, -v) = angle1.

Worldline Functions

Worldline functions use a given value to identify a point along the worldline and then return a corresponding value at that point. If no point can be identified, null is returned. If there are multiple matches, the first one is returned. To obtain a different match, apply an interval to the observer.

Identifying values are:

  • d: A distance in rest units, measured from whatever point is set to d = 0. Points before the 0 point are negative.
  • t: A rest time.
  • tau: Observer time, measured from whatever point is set to tau = 0.
  • v: Velocity relative to the rest frame.
Function Returns
dToT( float , observer ) Convert a distance to a rest time.
dToTau( float , observer ) Convert a distance to the observer's time.
dToV( float , observer ) Convert a distance to a rest frame velocity.
dToX( float , observer ) Convert a distance to an X position.
tToD( float , observer ) Convert a time to a distance.
tToTau( float , observer ) Convert a time to the observer's time.
tToV( float , observer ) Convert a time to a rest frame velocity.
tToX( float , observer ) Convert a time to an X position.
tauToD( float , observer ) Convert tau to a distance.
tauToT( float , observer ) Convert tau to a rest time.
tauToV( float , observer ) Convert tau to a rest frame velocity.
tauToX( float , observer ) Convert tau to an X position.
vToD( float , observer ) Convert a velocity to a distance.
vToT( float , observer ) Convert a velocity to a rest time.
vToTau( float , observer ) Convert a velocity to the observer's time.
vToX( float , observer ) Convert a velocity to an X position.

Bounding Box Functions

Adding a bounding box to a line affects how a line is drawn and whether an intersection with another line or a worldline occur.

Function Returns
setBounds( line , bounds ) Returns a line bounded by the bounding box. The original line is unaffected.
clearBounds( line ) Returns a line with any associated bounding box removed. The original line is unaffected.

Interval Functions

Adding an interval to an observer affects how the observer's worldline is drawn, whether an intersection ith a line occurs, and whether a given value can be used to identify a point along the worldline.

Function Returns
setInterval( observer , interval ) Returns an observer whose worldline lines within the interval. The original observer is unaffected.
clearInterval( observer ) Returns an observer with any associated interval removed. The original observer is unaffected.

Intersection Functions

If no intersection occurs, null is returned.

Function Returns
intersect( line , line ) Returns a coordinate indicating where the two lines intersect.
intersect( line , observer ) Returns a coordinate indicating the first point (earliest in time) where the line and observer intersect.
intersect( observer , line ) Returns a coordinate indicating the first point (earliest in time) where the line and observer intersect.

String Functions

Function Returns
toString( float , float ) Converts the first number to a string with the number of digits to the right of the decimal point being given by the second number. The function ignores the display and print precision set by the set statement.

Definition Functions

Function Returns
defined( variable ) Returns 1.0 if the variable has been defined and 0.0 if not.

Expressions
