Syntax Notation

  • Italics represent an element that is defined elsewhere.
  • Colored Bold represents an element that is entered exactly as it appears in the syntax.
  • The two items above are an "element."
  • Consecutive elements separated by spaces in the syntax can be separated by any amount of whitespace in the script or none at all.
  • Parentheses can be used to create an element from groups of elements.
  • A vertical bar ("|") signifes a choice: one of the elements.
  • An element followed by a question mark ("?") can appear 0 or 1 times (it's optional).
  • An element followed by an asterisk ("*") can be used 0 or more times.
  • An element followed by a plus sign ("+") can be used 1 or more times.
  • The definition for an element in italics may be given by listing the term followed by a right arrow ("â†’") and then the definition.

Note that scripts may be syntactically valid, but may still be invalid. The application will display an error when it encounters a problem.

You may encounter a something like: origin coordinate. The correct syntax would be origin expression. The notation should be interpreted as accepting a coordinate or any expression that evaluates to a coordinate.
