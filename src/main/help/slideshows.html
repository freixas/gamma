Slideshows

Slideshows are special types of scripts—ones that contain other scripts. Each script in a slideshow is called a slide.

Slideshows are intended for presentations such as lectures and for creating movies.

slideshow

slideshow → slideshow slideshowSettings? slideshowBlock
slideshowSettings → slideshowSetting ( , slideshowSetting )*
slideshowSetting → autoPlay | defaultPause float
slideshowBlock → { scriptStatement ( ; scriptStatement )* }

A slideshow script begins with the slideshow statement. This separates it from normal scripts.

A slideshow can contain two settings, both optional:

  • If autoPlay is entered, the slideshow begins in "play" mode, automatically advancing from one slide to the next. If autoPlay is not entered, the slideshow begins in "pause" mode, where slides advance only on request. The end user can change to "play" or "pause" mode at any time.
  • defaultPause must be followed by a non-negative number. This controls the default number of seconds to pause after a non-animated, non-interactive script finishes. Animated scripts always have a default pause time of 0—as soon as the last frame is displayed, the next slide is loaded. Interactive scripts, on the other hand (even animated ones), have an infinite pause time. The default pause time for any kind of script can be overridden in the script statement.

This slideshow statement is followed by a single block. This block contains only script statements.

Both slideshow and script statements, both of which are parsing-level statements. They can only contain literal values, not variables.

script

scriptStatement → script string scriptSettings?
scriptSettings → pause float

The script statement is folowed by a literal string that refers to a script file. If the file name is relative, it is relative to the location of the slideshow script. If a slideshow is located on the web, all scripts must come from the same domain. If the slideshow is on the local file system, each script can come from anywhere.

The optional pause controls the number of seconds to pause after a slide finishes. Unlike defaultPause, it applies to any kind of script. The pause time begins after the diagram is drawn or after the last frame of an animated diagram is drawn. Once the pause timer starts, it will be reset to its full pause time by any user interaction—zooming, panning, manipulating a GUI control, or changing the animation frame.

If a script within a slideshow is itself a script, it will be incorporated as though the individual scripts it contains had been entered in its place. The only difference is that the default pause times for scripts without explicit pause times will come from the nested slideshow rather than the top-level one.
