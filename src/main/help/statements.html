Comments

Scripts can contain two types of comments:

  • The character sequence "//" begins a comment that continue to the end of the line. Every character (including the "//") is removed before parsing the script.
  • The character sequence "/*" begins a comment that continues until the character sequence "*/" is found. Every character in the region (including the "/*" and "*/") is removed before parsing the script.

Overall Structure

A script is simply a sequence of statements.

Statements

A statement is the basic unit of a script. Statements generally end with a semicolon.

Blocks

A block is a statement that consists of a sequence of statements starting with { and ending with }. It does not end with a semicolon.

Parsing-level Statements

Parsing-level statements are statements that are handled by the parser, the code that turns the script into an executable chunk of code. Because the parser has limited capabilities, parsing-level statements cannot have variables or expressions.

include

include string

An include statement includes the text from another file into this one. The effect is as though those characters had been entered in place of the include statement.

The string is a filename. It can be a relative filename, in which case it's relative to the script file's location, or it can be an absolute filename.

stylesheet

stylesheet externall* string

The stylesheet statement adds a stylesheet to the diagram (see Stylesheets). If external is used, the stylesheet comes from a file and the string is the file's name. It can be a relative filename, in which case it's relative to the script file's location, or it can be an absolute filename.

If external is omitted, the entire stylesheet must be contained in the string. Since stylesheets can have values that are strings, you will need to be careful with how the internal strings are specified.

set

set setting ( , setting )*
setting → units float | displayPrecision float | printPrecision float

The set statement controls a few global settings:

  • units: The meaning of one unit in the diagram. A value of 1 means that units are in light years, x, and years, t. The choice of units affects the interpretation of acceleration values (which are in g's). If omitted, the value used is 1.
  • displayPrecision: The default precision for any number that is converted to a string and displayed on the diagram. The value represents the number of digits to the right of the decimal point. If omitted, the default value is 12.
  • printPrecision: The default precision for any number that is converted to a string and printed with the print statement.The value represents the number of digits to the right of the decimal point. If omitted, the default value is 12.

The override the precision, use the toString() function.

Assignment Statements

An assignment statement assigns a value to a variable. The value is derived from an expression. To learn about expressions and variables, see Expressions.

An assignment statement creates a variable if it doesn't exist.

Normal

leftVariable = expr
leftVariable → name | leftVariable . name

This is the normal form. The expression can evaluate to any type of value: a float, a string, or any object. The variable takes on the type of the value.

The period (".") used to dereference an object allows you to set the value of an object's field. In this case, the value must match whatever types are allowed by the object's field. For example, a frame object has two fields: v and origin. The former accepts a float while the latter accepts a coordinate.

static

static variable = expr

When a script is first loaded, there are no variables defined; variables are created as they are assigned their first value. Under some circumstances (for example, animation) a script may be re-executed without being re-loaded. For most variables, there is no difference between the first execution (when the script is loaded) and any subsequent execution—the script begins with a clean slate.

Static variables are one exception; once created, they retain their values across script executions (but not across script loads). Once a variable is marked as static, it remains static and can be used like any other variable.

To initialize a static variable only when it's undefined, use the defined() function in a pattern like this:

if (!defined(a)) {

static a = log(5);

}

You don't need to limit yourself to assigning a value only once. Nor do you need to mark a variable as static more than once (but it won't hurt if you do). This example shows how a static variable can function as a frame counter in an animation.

if (!defined(counter)) {

static counter = 0;

}

counter = counter + 1;

This counter would ignore such things as animation cycles and repetitions—it would increment with each re-execution of the script, no matter what caused the script to re-execute.

animate

animate leftVariable = float ( to float )? step float

The animate assignment creates an animation variable. When an animation runs, the script is executed once for each animation frame; the assignment, however, is ignored all but the first time. From then on, the value of the variable is controlled by the animation engine.

The value of the variable is incremented by its step size until its limit (the to value) is exceeded, if one is specified (if the step value is negative, the variable is decremented until it is less than the limit).

The first animation variable to exceed its limit ends a single repetition of an animation. If no animation variable has a limit, then the animation is theoretically infinite, although in practice there is a finite (but large) limit.

Animations only run when the animation command is present and there is at least one animation variable; otherwise, they operate like any other variable.

range

range leftVariable = float from float to float label string

The range assignment adds a slider to the diagram. The diagram is created by executing the script with the variable set to its assigned value. If the end user moves the slider, the script is re-executed. The assignment is ignored and the value of the variable is set to match the slider.

The slider's range is given by the second and third values. The initial value must be within the range.

The final string is a label displayed with the slider.

toggle

toggle leftVariable = float label string restart?

The range assignment adds a checkbox to the diagram. The diagram is created by executing the script with the variable set to its assigned value. If the end user toggles the checkbox, the script is re-executed. The assignment is ignored and the value of the variable is set to match the slider.

The initial value can be anything. It is interpreted as a boolean. The checkbox is checked if the value is not zero, unselected if it is 0. When the end user toggles it, the variable will be either 0 or 1, depending on whether the box is unchecked or checked.

The final string is a label displayed with the toggle.

restart causes all animation variables to be removed and the animation to restart. This allows the toggle to change the animation variables or the length of the animation.

choice

choice leftVariable = float choices string ( , string )* label string restart?

The choice assignment adds a choice button to the diagram. The diagram is created by executing the script with the variable set to its assigned value. If the end selects a different choice, the script is re-executed. The assignment is ignored and the value of the variable is set to match the choice number.

The choices are a list of strings. The first choice is number 1, the second 2, and so on. The initial value must match one of the choice numbers.

The final string is a label displayed with the choice button.

restart causes all animation variables to be removed and the animation to restart. This allows the end user's selection to change the animation variables or the length of the animation.

Conditional Statements

if

if ( boolean ) statement ( else statement )?

If the value following the if is true (non-zero), the first statement is executed. If it is false (0), the second statement, if present, is executed.

Looping Statements

while

while ( boolean ) statement

If the value following the while is true (non-zero), the statement is executed. This repeats until the value is false (0).

for

for variable = float to float step float statement

The variable is assigned the first value. It is then tested to see if it is greater than the second value (or less than, if the step value is negative). If the number passes the test, the statement is executed. The number is then incremented (or decremented) by the step value and the sequence repeats from the test.

break

break

The break statement causes an enclosing while or for loop to exit. A break statement outside any loop generates an error.

continue

continue

The continue statement returns an enclosing while or for statement to return to its conditional test. In the case of the for statement, the loop variable is incremented (or decremented if the step value is negative) before testing. A continue statement outside any loop generates an error.

Command Statements

Command statements are things that either control how the diagram is drawn or draw something on the diagram. The have a common syntax:

command ( name : expr ( , name : expr )* )?

The name/expression pairs are called properties and the sequence of properties is a property list. Each command has a set of properties that control how it behaves. Almost all properties are optional and have a default value if omitted. The specific properties supported by each command are defined below.

For some commands, there is one property that is either required or very common. This property is given without the property name or colon. In the tables below, these special properties are identified by a color: a red property is required and a green property is optional.

All commands have these three common stylesheet-related properties:

Name Type Default Min Max Description
id string ""     The id for this command.
class string ""     The stylesheet classes for this command. Each class name in the string should be separated by a comma.
style string ""     A set of styles. The braces around the styles are not needed. Within the string, multiple styles should be separated by semicolons.

For more details on these, see Stylesheets.

Control Commands

These commands can appear anywhere in the script, but they are executed before any drawing takes places. If used, each type should only appear once. If the same control command appears twice in a script, the last one is used and all others are ignored.

animation

For a diagram to be animated, the animation command must appear and there must be at least one animation variable defined.

Name Type Default Min Max Description
cycle boolean false     If cycle is false, one repetition of the animation is completed once the first animation variable with a limit exceeds its limit. If cycle is true, one repetition of the animation is completed once the first animation variable with a limit exceeds its limit and the frames are executed in reverse order.
reps int 500 1 500 The number of repetitions of the animation.
speed float 1 0.1 10 A multiplier applied to the “normal” speed (a value of 1 is “normal” speed, a value of 2 is twice as fast, and a value of 0.5 is half as fast).

display

The display command controls the overall appearance of the diagram. If no display command is given, a default one will be used. Since the display command's values are only used for the initial display, the values cannot be animated.

Name Type Default Min Max Description
width int int 1 Maximum display width The width of the diagram in pixels. If both width and height are omitted, the display area is always resized to the maximum space available. If only width is omitted, it is set to the maximum window width, but is not resized after that.
height int int 1 Maximum display height The height of the diagram in pixels. If both width and height are omitted, the display area is always resized to the maximum space available. If only height is omitted, it is set to the maximum window size, but is not resized after that.
units string "pixels"     The units for width and height. Valid values are "pixels", "inches", or "mm". The size of the image on the display will be correct for the units chosen only on the monitor on which the image is created. If moved to another monitor, the operating system may choose to retain the physical size at the expense of the pixel size. When the units are in pixels, the physical size of the image will vary on different monitors; if the units are in inches or millimeters, the size should be consistent, but the number of pixels will vary.
origin coord (50, 50)     The location of the origin of the default observer on the diagram. The display width is divided into 100 units such that 0 represents the left side of the diagram and 100 represents the right. The display height is also divided into 100 units where 0 is the bottom and 100 is the top of the diagram. Values are not limited to being from 0 to 100, though—the origin can be off-screen.
scale float 1 >0   Unscaled coordinates (i.e. pixels) are scaled by this factor to produce scaled coordinates. The scale is defined by taking the smaller of the diagram width or height. A scale of 1 means that this distance is exactly 100 units, a scale of 2 would be 50 units, a scale of .5 would be 200 units and so on.

frame

The frame command defines the inertial frame from whose perspective the diagram will be drawn. This is executed prior to each animation frame, so every animation frame can be drawn relative to a different inertial frame.

Name Type Default Min Max Description
frame frame The rest frame     The inertial frame from whose perspective the diagram will be drawn.

Drawing Commands

The drawing commands are executed in the order in which they appear in the script. If you want one thing drawn in front of another, the thing in front should appear later in the script.

While the properties control what is drawn, the stylesheet and styles control how it is drawn. See Stylesheets.

axes

axes frame propertyList

The axes command draws the axes for the given inertial frame. An observer may be substituted for a frame, in which case the observer's inertial frame at the observer's tau of 0 is used.

Name Type Default Min Max Description
frame frame The rest frame     The frame that defines the axes.
positiveOnly boolean false     Draw only the positive axes; otherwise, draw the full axes.
x boolean true     Draw the x-axis.
t boolean true     Draw the t axis.
xLabel string ““     The label for the x-axis.
tLabel string ““     The label for the t axis.

event

event coordinate propertyList

The event command places a marker at the given location (relative to the rest frame).

Name Type Default Min Max Description
location coord       The event location.
text String ""     A label for the text.
rotation angle 0.0     The rotation of the text in degrees around the anchor point. Positive rotation values rotate counter-clockwise. 0 is horizontal.
frameRelative-Rotation boolean false     If true, the rotation angle changes with the drawing frame. If false, it's frame-independent.
boostTo frame none     The frame to boost to. The location is boosted to this frame and a hyperbolic line joins the two positions. If boostTo is omitted, the selected event shape is drawn. If a frame is supplied, a hyperbolic line connects the location relative to the default frame and the location relative to the boost frame.

grid

grid frame propertyList

The grid command draws the grid for the given inertial frame. An observer may be substituted for a frame, in which case the observer's inertial frame at the observer's tau of 0 is used.

Name Type Default Min Max Description
frame frame The rest frame     The frame that defines the axes.
x boolean true     Draw the lines parallel to the x axis.
t boolean true     Draw the lines parallel to the t axis.

hypergrid

path

The path command draws a path.

  • If closed is true, the last point in the path is treated as connected to the first point.
    If stroke is true, the path outline is drawn.
    If fill is true, the path is treated as closed and the interior is filled.
Name Type Default Min Max Description
