Comments

Stylesheets can contain two types of comments:

  • The character sequence "//" begins a comment that continue to the end of the line. Every character (including the "//") is removed before parsing the script.
  • The character sequence "/*" begins a comment that continues until the character sequence "*/" is found. Every character in the region (including the "/*" and "*/") is removed before parsing the script.

Stylesheets

Commands control what is drawn; stylesheets control how it is drawn—the "look."

The stylesheet system is feature rich, allowing control of just about every aspect of the diagram's design, but it is not immediately essential to creating a diagram. Consider using the Stylesheet Quick Start at first.

A single diagram may contain multiple stylesheets. These are concatenated into a single stylesheet in this order:

  • The factory stylesheet
  • A global stylesheet selected through the Preferences dialog
  • Stylesheets defined using the stylesheet statement, in the order encountered

Stylesheets consist of a sequence of zero or more rules.

Rules

( selector ( , selector )* )? { ( styleProperty? ; )?* }

A rule consists of zero or more selectors, separated by commas, followed by a set of zero or more style properties enclosed within braces, with each style property ending with a semicolon.

The shorthand

selector1, selector2 { ... some style properties .. }

is exactly equivalent to

selector1 { ... some style properties .. }

selector2 { ... the same style properties ... }

Selectors

Selectors are used to select which style properties are applied to a given command.

There are three things used to match a selector to a command: the command name, the command's id value and one or more of the classes listed in the command's class value. Let's look at an example:

axes id: "id1", class: "class1, class2";

The three elements here are: the name "axes", the id "id1" and the two classes "class1" and "class2" (which are separated by commas). When used in a selector, the id value is prefixed with a "#" and each classname is prefixed with a ".". The elements desired for a match are then concatenated together. The following selectors would all match the command above:

  • axes,
  • #id1
  • .class2
  • axes#id1
  • axes.class1
  • axes#id1.class1
  • axes#id1.class.class2

This is not a complete list. For a match to occur, all parts must match. For example, axes.class1.class3 would not match.

A rule with no selectors matches everything.

Each match selector has a score. The score begins at 0, adds 1 for the command name, 10 for each class, and 1000 for an id match. The style properties associated with a matched selector are moved as a whole into a new list, with the highest scoring matches placed lower in the list. Within the same score, rules that appear later in the stylesheet are placed lower in the list.

To create the final set of styling values for a command, each style property is applied, starting from the top of the list.

Style Properties

Some style properties are "shorthands" for others. Setting one property may also cause others to be set, which may further cascade to others. For example, the color property will set the div-color property, which will then set the x-div-color, t-div-color, and major-div-color properties, the last of which sets two more. Therefore, setting major-div-color before color will have no effect.

The style property tables that follow indicate the inheritance hierarchy in the "Default" value column.

Color Values

There are three ways to name colors: using hexadecimal values, color names, or color functions.

Hexadecimal Colors

Hexadecimal colors use one byte each (written in hexadecimal) for red, green, blue, and alpha values, for a total of 8 hexadecimal characters. The 8 characters are preceded with a "#".

If there are only three or four characters, each characters is doubled (e.g. "#123" becomes "#112233"). If there are now only six characters, FF is appended (an alpha value that means fully opaque).

Color Functions

There are four color functions:

  • rgb(): This takes three values, for red, green, and blue. The values can range from 0 to 255 or 0 to 100%.
  • rgba(): This takes four values. The first three are the same as for rgb(). The fourth is an alpha value from 0.0 to 1.0.
  • hls(): This takes three values. The first is a hue, from 0 to 360, with red being 0 (or 360). The second values range from 0 to 100% and are saturation and lightness.
  • hsla(): This takes four values. The first three are the same as for hsl(). The fourth is an alpha value from 0.0 to 1.0.

Default Stylesheet

Since most style properties default to the values given above, the default stylesheet is mainly useful for command-specific defaults.

grid { color: #CCC; }

hypergrid { color: #3D3; }

worldline { color: #00F; }

axes { font-size: 14; tick-font-size: 10; }
